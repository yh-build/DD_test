연안침식 드론 측량 SaaS

상단 메뉴 기반으로 관리자 업무와 공통사용자 업무를 나누는 1단계(PoC·PoM) 화면 설계안입니다.

관리자 관리자 홈 프로젝트 관리 분석 현황 리포트 사용자·권한 내 계정
실제로 클릭해 보는 목업은 07_SaaS_작업자_목업.html · 08_SaaS_관리자_목업.html 입니다. 이 문서는 두 화면의 메뉴 구조와 흐름을 설명하는 구성안입니다.

메뉴 구조

메뉴는 적게 유지하고, 하위 동작은 각 페이지 안의 카드·탭·드릴인으로 처리합니다.

역할메뉴핵심 목적
작업자현장 홈배정된 프로젝트의 최근 회차·업로드 상태·오차·지연·판정 확인
드론 측량현장 기준 등록·측량 설정(GCP·검사점) → 회차 생성·업로드 → 정합 시각화(GCP 매칭) → 재촬영 판정(현장 즉시판정)
분석배정 현장의 원본 처리·정밀 정합·최종 5cm 검증 조회
리포트프로젝트 목록에서 진입 → 회차별 결과 문서 조회·다운로드
내 계정본인 정보 + 현재 비밀번호 확인 후 재설정(2단계)
관리자관리자 홈전체 프로젝트·현장·회차·관리 필요 항목 대시보드
프로젝트 관리프로젝트 목록에서 생성·진입 → 현장·기준자료 검수 → 참여자 배정 (현장 등록 자체는 작업자도 수행)
분석 현황전체 프로젝트 분석 + 시스템 성능지표(5cm·3초)와 내부 성공률 달성 집계
리포트전체 프로젝트 리포트 조회·다운로드(프로젝트/현장/회차 단위)
사용자·권한회사 접속 가입 요청 승인 + 구성원 관리(프로젝트 권한은 프로젝트 관리에서)
내 계정본인 정보 + 비밀번호 재설정(2단계)

권한 모델

가입은 셀프, 회사 접속은 관리자 승인, 프로젝트 접근은 프로젝트별 부여 — 3단계로 분리합니다.

1
회사 코드로 가입 요청 사용자가 로그인 화면에서 회사 코드로 직접 가입을 요청합니다. (셀프)
2
회사 접속 승인 관리자가 사용자·권한에서 가입 요청을 승인하면 회사 구성원이 됩니다. 거부·비활성화도 여기서.
3
프로젝트 권한 부여 관리자가 프로젝트 관리 > 참여자 배정에서 구성원을 특정 프로젝트에 추가합니다. 한 사람이 여러 프로젝트 가능.

프로젝트 구조와 기준자료

데이터 계층
프로젝트 → 현장 → 회차 → 소티 → 촬영
회차는 배터리 교체로 여러 소티(비행 차수)로 나뉘고, 소티에 촬영(건)이 담깁니다. 커버리지·원본은 소티를 합산, 3초 지연은 촬영 건별로 봅니다.
기준자료 위치
현장=고정 기준 · 회차=측량
현장에는 좌표계·기준점(수준점) 성과·반영구 고정점·기준 정사영상/DSM·특징점만 버전 등록. 회차별 GCP·검사점 실측은 드론 측량 시 입력합니다.
회차와 기준자료
촬영 당시 버전 참조
회차는 생성 시점의 기준자료 버전을 참조해 재현성을 확보합니다.

데이터 2-경로 (PoC)

계획서 기준 — 경량은 실시간 전송해 현장에서 즉시 판정, 원본은 드론 SD에 두었다가 착륙 후 비동기 처리. (도킹스테이션·자동비행은 2단계, PoC 제외)

① 경량 · 실시간 3초 / 촬영 건별

저해상 영상 + 특징점 + RTK 위치만 LTE/5G로 전송 → 서버 실시간 정합 → 현장 단말에 결과 푸시. 풀해상도 원본은 보내지 않음.

전송 데이터저해상 영상 · 특징점 · RTK (원본 제외)
지연 측정데이터 수신 ~ 결과 전송 ≤ 3초, 촬영 건별·반복(10·20회)
산출추정 정합 오차 · 재촬영 1차 판정 (현장 즉시)

② 원본 · 비동기 착륙 후

원본(풀해상도)은 드론 SD에 보관 → 착륙 후 업로드 → 비동기 정밀 처리. 실시간 3초 대상 아님.

업로드현장 PC/콘솔에서 chunk·resume 업로드
정밀 처리GCP 매칭·번들조정 → 검사점 대비 5cm 최종 검증
후처리정사영상 · DSM(·DTM) 산출 — 원본 받으면 처리(비동기)

원본 업로드·검증 콘솔 흐름

PoC 웹 콘솔: 회차·소티의 촬영 파일을 올려 빠른 검증 후 원본을 이어 올립니다. (드론 측량 화면)

1
회차·소티 생성회차 명칭(조석·고도) + 소티(비행 차수) 생성
2
파일·폴더 선택현장 PC에서 소티 촬영 파일·폴더 선택
3
업로드 시작경량 패키지 생성·전송 시작
4
경량화축소 이미지·위치 로그·구성 내역 생성(진행 표시)
5
빠른 검증조사범위·누락 영역·재촬영 1차 판단(진행 표시)
6
원본 업로드백그라운드 전송, 브라우저 닫히면 재개 대기

원본 업로드 완료 후 서버에서 정밀 정합 → 검사점 5cm 검증 → (비동기) 정사영상·DSM 산출이 수행됩니다 (아래 처리 파이프라인 참조). 결과는 분석 화면에서 확정됩니다.

처리 파이프라인 원본 · 비동기

원본 업로드 후 비동기로 수행되는 정밀 처리입니다. 현장 실시간 1차 판정(경량·3초)과는 별개이며, 여기서 5cm·정사영상·DSM이 확정됩니다.

1
현장 측량 + 수준점 처리GCP·검사점 RTK 실측 → 타원체고를 지오이드·국가수준점으로 보정해 정표고·국가좌표 확정. 같은 날 연속 비행은 GCP 동일(재사용), 날짜가 바뀌면 재측량.
2
영상 정합 · 공중삼각측량특징점(tie point) 매칭 + 영상에서 GCP 검출(코드 타겟 자동 / 일반 마커는 수동 시드 후 자동) → 실측 GCP와 매칭 → 번들조정으로 절대좌표·카메라 자세 확정 (검사점 RMSE ≤ 5cm)
3
조밀 점군 → DSM조밀 점군 생성 후 격자화 → 수치표면모델(DSM). 필요시 지면만 남겨 DTM
4
정사보정 → 정사영상DSM·카메라 자세로 기복·기울기 변위 제거 → 지도좌표 정사영상(Orthomosaic) 모자이크
5
검증 · 변화탐지검사점으로 5cm 검증, 기준 정사영상/DSM과 비교해 연안침식 변화탐지

의존 관계: 정사보정은 DSM이 있어야 가능(높이로 기복 변위 보정)하고, DSM은 GCP 매칭·번들조정으로 절대좌표가 확정된 뒤 만들어집니다. 표준 산출물: 정사영상 · DSM(·DTM) · 점군 · 오차 리포트.
